# LP 全面リデザイン計画

## 作成日: 2026-03-22

---

## 1. 現状の問題（なぜ根本的に変えるのか）

### 構造の問題
- **同じ内容を3回繰り返している**: Problem / Solution / Features が全て「匿名・仲間・記録」
- **9セクションは多すぎる**: ユーザーは途中で離脱する
- **全セクションが「見出し→カード3枚」**: 単調で物語性がない

### ビジュアルの問題
- **SaaSテンプレートの文法**: ブラーオーブ、グラスモーフィズム、グラデーションテキスト → 依存症回復アプリの「温かさ・安全感」を表現できていない
- **感情に訴えるビジュアルがゼロ**: テキストとカードだけ
- **スクリーンショットが二重フレーム**: CSSフレーム内にフレーム付き画像

### コンテンツの問題
- **実績の数字を全く活かしていない**: 7,900+ユーザー、90,000+投稿、130万+いいね — なのにLPは「28+レビュー」しか見せていない
- **28件の実レビュー中3件しか使っていない**: 最強の証言が埋もれている
- **ソーシャルプルーフの位置が遅い**: ページ下部。Hero直後に置くべき
- **「許可」の言葉がない**: 「失敗しても大丈夫」「完璧じゃなくていい」— レビューでは最も評価されているポイントなのにLPに反映されていない

## 2. 参考にした最新事例（2025-2026急成長アプリ）

| アプリ | 学ぶべきポイント |
|---|---|
| **Hallow** (祈り/瞑想) | ダーク×バーガンディの温かさ。リアルタイムカウンター「190万人が今祈っている」。キャンペーン型の緊急性 |
| **Opal** (スクリーンタイム) | 問題の直面化（「1日5-6時間スマホ」）→ 解決策。ダークネイビー+グラデーション。ライブカウンター |

| **Rise** (睡眠) | 科学的根拠で信頼構築。「80%が5日以内に効果実感」。グラデーション球体のビジュアルアンカー |
| **Headspace** (瞑想) | カスタムフォント。クリーム+ブルー。カテゴリ選択型Hero。「Try for $0」のリスク除去CTA |
| **Notion** (プロダクティビティ) | ブランドキャラクター。物語型のセクション構成。セクションごとに異なるレイアウト |
| **Raycast** (ランチャー) | 名前付きインフルエンサーの証言壁。カードごとに異なるグラデーション |
| **Strava** (フィットネスSNS) | 「Community-Powered Motivation」— 2語でカテゴリを再定義 |

### テンプレート感を消す5つの条件（調査結果）

1. **カスタムフォント or 特徴的なフォントペアリング** — 最もインパクトが大きい
2. **物語構造** — 機能リストではなく感情のアーク（問題→解決→証明→帰属）
3. **ドメイン固有の言語** — 汎用ウェルネス語彙ではなく、回復者が実際に使う言葉
4. **セクションごとに異なるレイアウト** — 「見出し+カード3枚」の繰り返しを避ける
5. **雰囲気を作る背景デザイン** — 背景がブランドそのもの（Hallowのバーガンディ、Opalのグラデーション）

## 3. 新しいデザイン方針

### 3-1. トーン & 感情

**目指す感情**: 「ここなら安心して話せる」「一人じゃないんだ」「失敗しても大丈夫」

- Hallowの「静かな温かさ」
- Gentler Streakの「罪悪感の解放」
- Stravaの「コミュニティの力」

### 3-2. カラーパレット

**方向性**: 温かみのあるニュートラル + テーマカラーアクセント

```
ベース（ライトモード）:
  背景:     #FAFAF8 (ウォームホワイト)
  カード背景: #FFFFFF
  テキスト:  #1A1A1A (ほぼ黒)
  サブテキスト: #6B6B6B

ベース（ダークモード）:
  背景:     #0F0F0F
  カード背景: #1A1A1A
  テキスト:  #F5F5F3
  サブテキスト: #999999

アクセント（テーマカラー — ネームスペースで変動）:
  デフォルト: #2E7D6F (ティールグリーン — 信頼・成長)
  CTA:      #E8734A (コーラル — 温かい行動喚起)

ネームスペース別:
  alcohol:  既存ベース調整
  kinshu:   既存ベース調整
  porn:     ダークテーマ維持、パープルアクセント調整
  tobacco:  既存ベース調整
```

### 3-3. タイポグラフィ

```
見出し: Inter (weight 700-800, letter-spacing: -0.03em)
  → タイトな字間で現代的な印象に
  → clamp() でフルード対応

本文: Noto Sans JP / Inter
  → 現状維持、読みやすさ優先

見出しサイズ:
  Hero h1:    clamp(2rem, 5vw, 3.5rem)
  セクション h2: clamp(1.5rem, 3vw, 2.25rem)
  カード h3:   clamp(1.125rem, 2vw, 1.375rem)
```

### 3-4. アニメーション

**方針**: 目的のあるマイクロインタラクションのみ。装飾的なアニメーションは排除。

- **スクロール表示**: fadeInUp (CSS scroll-timeline, 150ms ease-out)
- **ホバー**: scale(1.02-1.03) + 150ms transition
- **カード**: 軽い影の変化のみ（グラスモーフィズムは使わない）
- **`prefers-reduced-motion`**: 全アニメーション無効化

**削除するもの:**
- ブラーオーブアニメーション
- グラスモーフィズムカード
- パルスグローCTA
- ドットグリッドパターン
- ウェーブディバイダー


### ブラーオーブの代わりに何を使うか

装飾CSSを全削除すると平坦になりすぎる。代わりに以下で「雰囲気」を作る：

1. **セクション背景色の交互切り替え** — Gentler Streak方式。セクションごとに微妙に色が変わることでリズムが生まれる
2. **タイポグラフィの力** — Hero見出しを大きく太く（clamp(2rem, 5vw, 3.5rem), weight 800）。Superhuman/Arc方式で、文字自体がビジュアル要素
3. **数字の大きさ** — Social Proof Barの数字をfont-size 3rem以上で表示。BetterHelp/Hallow方式
4. **余白の贅沢さ** — セクション間のpaddingを広く取る（py-24〜py-32）。プレミアム感は余白から生まれる
5. **レビューカードの引用符デザイン** — 大きな「"」マークで引用感を演出。テキストだけで温かみを出す

### 今回のPhase 1で対応: スクリーンショット二重フレーム問題

現状: CSSデバイスフレーム内にフレーム付き画像 → 二重フレーム
対応: CSSフレームを簡素化 or 削除して、画像をそのまま角丸+影で表示
→ 新スクリーンショットがなくても見た目は改善される
