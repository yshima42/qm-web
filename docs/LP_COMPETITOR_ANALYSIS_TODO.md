## 現状のQuitMate LPの問題点まとめ

### 構造の問題

1. **セクションが多すぎる（9個）** — Hero, Problem, Solution, Features, HowItWorks, SocialProof, FAQ, FinalCTA, Footer
2. **Problem / Solution / Features が同じレイアウト** — 全部「見出し → カード3枚」の繰り返し
3. **Solution（3つの柱）と Features（3つの特徴）の内容が重複** — 匿名/仲間/記録 を2回言っている

### 視覚の問題

4. **スクリーンショットが壊れている** — フレーム付き画像をCSSフレームに入れて二重フレーム
5. **背景画像・イラストがゼロ** — テキストとカードだけ。感情に訴えるビジュアルがない
6. **CSSエフェクトがテンプレート感を消せていない** — ブラーオーブ、グラスモーフィズムを足しても根本構造が同じだから効果薄い

### コンテンツの問題

7. **ソーシャルプルーフが弱い** — 28件の実レビュー中3件しか使っていない
8. **ソーシャルプルーフの位置が遅い** — ページ下部にある。Hero直後に置くべき

## 新しい設計案（調査後に最終確定）

### 構成案: 7セクション

```
1. HERO
   - 共感型の大きな見出し（グラデーションテキスト）
   - サブ見出し
   - CTA（ストアバッジ）
   - ★4.9 ミニバッジ
   - アプリモックアップ（フレームなしスクリーンショットに差し替え後）

2. SOCIAL PROOF（Hero直後 — 早い段階で信頼確立）
   - 数字ハイライト（4.9評価、28+レビュー、12+カテゴリ）
   - メディアロゴ（掲載実績があれば）
   - または実レビューのハイライト引用1-2件

3. FEATURES（左右交互レイアウト — カード3枚繰り返しではない）
   - 機能1: 左テキスト / 右スクリーンショット
   - 機能2: 右テキスト / 左スクリーンショット
   - 機能3: 左テキスト / 右スクリーンショット
   ※スクリーンショット差し替え後に実装。差し替え前は3カラムカードのまま

4. HOW IT WORKS（3ステップ）
   - ダウンロード → 悩みを選ぶ → 仲間と回復

5. REVIEWS（大量のリアルレビュー）
   - App Storeから5-6件の実レビューを表示
   - docs/app-reviews.md から最強の引用を選出
   - カルーセルまたはメイソンリーグリッド

6. FAQ（4問）

7. FINAL CTA + Footer
```

### 削除するもの
- **Problem** — 共感はHeroの見出しに統合（「共になら、やめられる」で十分）
- **Solution** — Featuresに統合（重複排除）

### 保留（スクリーンショット差し替え後）
- Features の左右交互レイアウト
- Hero のモックアップ改善

## 画像素材の取得方法

### 選択肢

1. **アプリのスクリーンショット（フレームなし）**
   - Xcode Simulator → Cmd+S でスクリーンショット保存
   - 実機 → 電源+音量上ボタン → 写真アプリからエクスポート
   - ステータスバーを含めた純粋な画面キャプチャ

2. **フリー素材（背景画像用）**
   - Unsplash: https://unsplash.com/s/photos/recovery-hope
   - Pexels: https://www.pexels.com/search/wellness/
   - キーワード: recovery, hope, sunrise, nature calm, peaceful morning

3. **AIイラスト生成**
   - Midjourney / DALL-E でカスタムイラスト
   - BetterHelp風の多様な人々のイラスト
   - Headspace風の抽象的ウェルネスイラスト

4. **CSSのみで表現可能な視覚要素（画像不要）**
   - メッシュグラデーション背景
   - ブラーオーブ
   - ドットグリッドパターン
   - ウェーブディバイダー
